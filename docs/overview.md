# Invoice Parser — Quick Overview

**Invoice Parser** (this repo) is an internal-style web app for **ingesting telecom/carrier invoices** (PDFs and images), **extracting** summary fields, charges, usage, and inventory-style rows, letting someone **review and fix** the result, then **saving** everything to a **SQL Server** database. It learns over time: corrections and feedback can update **regex rules** and drive **ML retraining**, with optional hooks to a **separate Python service** (OCR + LayoutLM-style models) and optional **HTTP-based LLM** assist for hard cases—so the “product” is really **document capture + validation + continuous improvement**, not billing settlement or customer portals.

**Technically** the core is **.NET 7**: **ASP.NET Core MVC** (Razor) for the UI, **Entity Framework Core 7** against **SQL Server**, split into an **application database** (invoices, rules, feedback) and a **reference database** (customers and carriers). Parsing on the C# side uses **PDF and image pipelines** (e.g. **PdfPig**, **Tesseract** OCR), **Microsoft.ML** for field prediction, and vendor-specific logic (e.g. Verizon-oriented handling). A **FastAPI + Python 3.10+** sub-project adds **PyTorch/transformers**-based extraction when enabled; configuration wires it in over HTTP. There is also a minimal **JSON parse API** on the same app for programmatic uploads.

**Maturity caveat:** This is best thought of as a **strong MVP / early internal platform**: the architecture is a **modular monolith** with clear **Core / Infrastructure / Web** layers, but **automated tests and CI** are not prominent in the repo, and **auth and secret handling** need to be treated as deployment concerns—you’d secure and operate it like any app that touches **PII-heavy financial documents** and optional **external AI** calls.
